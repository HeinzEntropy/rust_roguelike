# 项目名Rust Roguelike
## 来源
Rust游戏开发实战，本项目旨在复刻学习该书中的，远期目标意图实现自己所希望的内容
## 剧情
要去找宝贝，才能拯救世界，期间的关卡是随机生成的
## 游戏流程
1. 进入地下城
2. 探索地图
3. 与敌人战斗，要么击败敌人，要么被杀死结束游戏
4. 寻找道具提升实力，道具内容待定
5. 寻找出口，并返回步骤1
## 最简可行产品（Minimum Viable Product，MVP）
1. 创建一个基础的地下城地图； 
2. 将玩家角色置于地下城之中，并让它可以在地下城中走动； 
3. 生成怪兽，将它们绘制在屏幕上，并且让玩家与怪兽相遇，进而杀死怪兽； 
4. 为玩家添加生命值，并且添加基于生命值的战斗系统； 
5. 添加可以恢复生命值的药品； 
6. 当玩家角色死亡时，游戏结束画面将显示在界面上； 
7. 在关卡中添加“亚拉的护身符”，并且让玩家角色在碰到它的时候取得游戏的胜利。
## 延展目标
1. 添加视野机制； 
2. 添加更有趣的地下城设计； 
3. 添加更多的地下城主题风格； 
4. 使地下城变成多层的结构，并且把护身符放在最深的一层； 
5. 为游戏添加各式各样的武器； 
6. 改为使用数据驱动的设计模式来生成敌人； 
7. 考虑引入一些视觉效果，以使战斗效果更加酷炫； 
8. 考虑实现计分机制。 
9. 添加出生点基地，玩家可以在基地进行准备工作
10. 添加轮回系统，每次轮回将收集到的道具带回基地
## 敌人
### 绿色妖精
- 它是小型的类人生物，所以需要具有和其他类人生物一样的 Re der 组件。
- 它在地图上具有 Position 组件。
- 它是易怒的，因此需要一个AI组件来使它在看到玩家角色时发起攻击。
- 它喜欢远距离攻击，因此AI组件需要在其逻辑上体现出这一点。
- 它们非常胆小，如果开发者想实现一个让它们四散逃跑的效果，那么可能需要引入一个新的组件。
- 它们很脆弱，意味着它们具有 Health 组件，而且其中存储的生命值很低。
